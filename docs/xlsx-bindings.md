 # 异环云配队.xlsx 绑定清单（表更新必看）
 
 目标：当 xlsx 更新导致单元格位置/范围变化时，网页仍能正确读取输入、生成下拉选项、驱动计算并取回结果。需要保证下列“绑定点”稳定，或在更新后同步调整网页侧绑定规则/寻址逻辑。
 
 ## 1. 必须存在的 Sheet 名称
 
 - 云配队（主界面输入与部分输出）
 - 配装（可编辑配置页）
 - 动作（角色→动作列表）
 - 增益（buff列表）
 - 中间（下拉/中间输出，网页用来生成选项或参考）
 - 计算（核心计算与列组结构）
 - 角色、天赋、弧盘、卡带（计算依赖数据源）
 
 如果 Sheet 改名，需要同时更新前端读取逻辑（目前是通过固定 sheetName 访问）。
 
 ## 2. 主界面（云配队）输入绑定
 
 ### 2.1 角色选择
 
 - 角色下拉输入单元格：F4:F7
 - 角色候选来源：配装!A3:A18（或等价的角色列表列）
 
 UI：左侧 4 个角色下拉 + 头像显示。
 
 ### 2.2 动作位（横向可扩展）
 
 - 动作选择区域（4 行 × N 列）：
   - 第 1 行：J4 起向右
   - 第 2 行：J5 起向右
   - 第 3 行：J6 起向右
   - 第 4 行：J7 起向右
 - 动作位权重/重复（1 行 × N 列）：J8 起向右
 
 UI：点击格子弹出下拉，候选随角色变化。
 
### 2.2.3 动作伤害（只读显示）

- 动作伤害行：J9 起向右

说明：该行是表内计算输出（含 FILTER 等公式），网页侧只读展示用于辅助查看。

### 2.2.1 时间刻度（只读显示）

- 时间刻度行：J10 起向右（横向与动作位对齐）

UI：动作区上方的时间刻度，仅用于显示（不编辑）。

### 2.2.2 反应与倾陷状态（横向与动作位对齐）

- 反应：第 11 行（J11 起向右）
- 倾陷状态：第 12 行（J12 起向右）

UI：位于动作区与 buff 区之间，可单击下拉与框选批量编辑。

 ### 2.3 Buff 位（横向与动作位同列对齐）
 
 - 自身 buff：第 14、15 行（J14 起向右）
 - 全队 buff：第 16~21 行
 - 卡带 buff：第 22~25 行
 - 弧盘 buff：第 26~29 行
 
 buff 候选来源（用于 UI 下拉）：
 - 增益!A2:A11（自身 buff）
 - 增益!A12:A22（全队 buff）
 - 增益!A23:A27（卡带 buff）
 - 增益!A28:A39（弧盘 buff）
 
 ### 2.4 其他配置（当前 UI 暴露项）
 
- 指定轴长：表内“指定轴长”标签下一行的数值单元格（默认布局为 AD13；扩列后该区域会右移，网页按标签定位写入）
 - 角色等级：F9
 - 怪物等级：G9
 - 倾陷上限：H9
- 怪物弱点：G11（枚举建议：无/光/灵/咒/暗/魂/相）
 - 怪物抗性：F11
 - 弱点减抗：H11
 - 对敌类型：G13（枚举：群怪/BOSS）
- 早雾DOT层数：H13
 
 如果这些配置位置变化，需要同步更新绑定。
 
 ## 3. 配装页（配装）输入绑定
 
 每个角色一行，行号区间默认 3~18（A 列为角色名）。
 
 - 角色名：A{row}（建议作为主键；更新表时尽量保持角色名文本稳定）
 - 适配类型：C{row}（用于弧盘下拉过滤）
 - 弧盘：V{row}
 - 卡带：X{row}
 - 卡带主词条：AB{row}
 - 主词条数值：AC{row}
 - 副词条（数值型）：AG~AQ{row}
 - 觉醒开关（布尔）：AR~AW{row}
 
 备注：如果后续希望暴露“空幕属性/被动/层数”等字段，需要追加：
 - 空幕属性：AD{row}
 - 空幕被动：AE{row}
 - 空幕被动层数：AF{row}
 
 ## 4. 动作列表绑定（动作）
 
 UI 的动作下拉不直接依赖 Excel 的数据验证，而是按“动作”sheet计算得到：
 
 - 动作!A 列：角色名
 - 动作!B 列：动作名
 - 默认有效区间从第 4 行开始（A4/B4 起向下）
 
 若列含义或起始行变更，需要同步更新读取规则。
 
 ## 5. 弧盘/卡带候选绑定
 
 - 卡带候选：卡带!A3 起向下（到空为止）
 - 弧盘候选：
   - 弧盘!A 列：弧盘名
   - 弧盘!F 列：适配类型（用于按 配装!C{row} 过滤）
 
 若“弧盘下拉”改为别的逻辑（例如按角色/元素过滤），需要同步调整网页的过滤规则。
 
 ## 6. 计算组扩展绑定（计算）
 
 网页支持把“计算”sheet的列组从有限数量扩展为更多组（动作位数量 N）。
 
 依赖假设（更新表时尽量保持）：
 - “计算”sheet 第 1 行存在整数 1、2、3… 作为每个动作位列组的起始标记
 - 相邻两个组起始标记的列差 = 组宽度（当前表为 29 列/组）
 - 组内公式通过 “INDEX(云配队!$Jx:$ACx, 1, <组起始列>$1)” 形式用组号索引动作位
 
 如果表作者改变了组宽度、组号位置、或把索引方式从 INDEX 改为别的函数，需要同步调整“扩展算法”。
 
 ## 7. 结果绑定（云配队）
 
 为避免列扩展导致输出单元格列号变化，网页通过“文本锚点”寻址：
 
 - 在 云配队 中搜索文本 “DPS”，取其下一行同列作为 DPS 输出单元格
 - 搜索文本 “累计伤害”，取其下一行同列作为总伤害输出单元格
 - 搜索文本 “贡献伤害”，取其所在列并读取下方 4 行作为 4 个角色贡献伤害
 
 若这些中文/英文锚点文案改动，需要同步更新寻址锚点。

## 8. 动作占比图数据源（中间）

主界面结果区额外展示“首角色动作占比”（柱状图），数据源为：

- 中间!H3:H17：动作名（含 FILTER 结果与固定动作名）
- 中间!I3:I17：对应动作占比/权重（SUMIF 计算结果）

说明：
- 网页侧直接读取“重算后结果”并计算百分比展示，不依赖 Excel 的数据验证规则。
